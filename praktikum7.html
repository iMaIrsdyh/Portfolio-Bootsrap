Ringkasan Praktikum 7

Laporan lengkap migrasi database, data seeding, routing resource, model Eloquent, pengendali resource, dan tampilan Blade untuk entitas Product, Customer, dan Mahasiswa.

Migration & Seeding

Membangun struktur tabel dan memasukkan data awal untuk pengujian cepat aplikasi.

Routing Resource

Memanfaatkan resource route Laravel untuk menghasilkan route CRUD secara otomatis.

Blade View

Menampilkan data dengan struktur halaman yang konsisten dan profesional menggunakan template Blade.

1. Konfigurasi Database

Tahap awal praktikum adalah membangun koneksi database yang stabil. Database praktikum_laravel dibuat melalui phpMyAdmin XAMPP dan kemudian disambungkan ke aplikasi Laravel di file .env.

DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=praktikum_laravel
DB_USERNAME=root
DB_PASSWORD=

Setelah konfigurasi selesai, Laravel berhasil terhubung dengan MySQL sehingga perintah migrasi dan seeding dapat dieksekusi.

Hasil

Koneksi database aktif, lingkungan pengembangan siap, dan struktur migrasi dapat dibuat tanpa error.

2. Migration

Migration digunakan untuk mendefinisikan struktur tabel dengan kode sehingga perubahan dapat dilacak dan direset.

2.1 Migration Product

Membuat tabel products untuk menyimpan data barang. 

php artisan make:migration create_products_table

Isi migration:

Schema::create('products', function (Blueprint $table) {
    $table->id();
    $table->string('name');
    $table->integer('price');
    $table->text('description');
    $table->timestamps();
});

2.2 Migration Customer

Membuat tabel customers untuk menyimpan informasi pelanggan.

php artisan make:migration create_customers_table

Isi migration:

Schema::create('customers', function (Blueprint $table) {
    $table->id();
    $table->string('nama');
    $table->string('email')->unique();
    $table->string('telepon');
    $table->text('alamat');
    $table->timestamps();
});

2.3 Migration Mahasiswa

Membuat tabel mahasiswas sebagai data akademik mahasiswa.

php artisan make:migration create_mahasiswas_table

Isi migration:

Schema::create('mahasiswas', function (Blueprint $table) {
    $table->id();
    $table->string('nim')->unique();
    $table->string('nama');
    $table->string('jurusan');
    $table->string('email')->unique();
    $table->integer('angkatan');
    $table->timestamps();
});

Menjalankan Migration

Setelah seluruh migration selesai dibuat, perintah berikut dieksekusi untuk membuat tabel di database:

php artisan migrate
Hasil

Berhasil membuat tabel products, customers, dan mahasiswas di database praktikum_laravel tanpa error.

3. Seeding

Seeder digunakan untuk mengisi database dengan data dummy sehingga aplikasi dapat diuji tanpa masukan manual.

3.1 Product Seeder

Membuat dua data produk awal untuk pengujian daftar produk.

php artisan make:seeder ProductSeeder

3.3 Mahasiswa Seeder

Class seed sederhana untuk data mahasiswa tunggal sebagai contoh entitas akademik.

php artisan make:seeder MahasiswaSeeder

Isi seeder:

DB::table('mahasiswas')->insert([
    [
        'nim' => '2411533018',

Daftarkan Seeder

Seeder didaftarkan di file database/seeders/DatabaseSeeder.php agar dapat berjalan bersama. 

$this->call(ProductSeeder::class);
$this->call(CustomerSeeder::class);
$this->call(MahasiswaSeeder::class);
Perintah Jalankan Seeder
php artisan db:seed

Database terisi otomatis dengan data awal sehingga fitur read dan list dapat diuji.

4. Model

Model Eloquent menghubungkan tabel database dengan aplikasi sehingga data dapat diolah secara objek.

4.1 Product Model

php artisan make:model Product

Tambahkan atribut yang dapat diisi massal.

protected $fillable = [
    'name',
    'price',
    'description'
];

4.2 Customer Model

php artisan make:model Customer

Konfigurasi mass assignment untuk entitas pelanggan.

protected $fillable = [
    'nama',
    'email',
    'telepon',
    'alamat'
];

4.3 Mahasiswa Model

php artisan make:model Mahasiswa

Menentukan kolom yang boleh diisi saat CRUD Mahasiswa dijalankan. 

protected $fillable = [
    'nim',
    'nama',
    'jurusan',
    'email',
    'angkatan'
];

5. Routing

Routing menghubungkan URL dengan controller. Resource route Laravel menghasilkan semua route CRUD dalam satu deklarasi.

Route::resource('products', ProductController::class);
Route::resource('customers', CustomerController::class);
Route::resource('mahasiswa', MahasiswaController::class);

Route resource membuat method index, create, store, show, edit, update, dan destroy secara otomatis.

Keuntungan
  • Mengurangi duplikasi route
  • Standarisasi naming convention
  • Mudah dikelola pada aplikasi CRUD

6. Controller

Controller bertanggung jawab menerima permintaan, memproses model, dan mengirim data ke tampilan.

6.1 Product Controller

php artisan make:controller ProductController --resource

Contoh method index() untuk mengambil semua product. 

public function index()
{
    $products = Product::all();

    return view('products.index', compact('products'));
}

6.2 Customer Controller

php artisan make:controller CustomerController --resource

Contoh method index() untuk menampilkan daftar customer.

public function index()
{
    $customers = Customer::all();

    return view('customers.index', compact('customers'));
}

6.3 Mahasiswa Controller

php artisan make:controller MahasiswaController --resource

Contoh method index() untuk menampilkan semua mahasiswa.

public function index()
{
    $mahasiswas = Mahasiswa::all();

    return view('mahasiswa.index', compact('mahasiswas'));
}

Fungsi Tambahan

Selain index(), controller resource biasa juga berisi store(), update(), dan destroy() untuk operasi CRUD penuh.

Hasil

Kelas controller mampu mengelola aliran data antara model dan view sesuai standar MVC Laravel.

9. Kesimpulan

Praktikum 7 memperkuat prinsip MVC Laravel dengan membuat layer database, model, controller, dan view secara terpisah namun terintegrasi. Penggunaan migration dan seeder membantu manajemen data, sedangkan resource route dan controller resource mempermudah alur CRUD.

Implementasi pada entitas Product, Customer, dan Mahasiswa menunjukkan bahwa Laravel dapat memfasilitasi pengembangan aplikasi web terstruktur dengan Eloquent ORM dan Blade Template Engine.

Hasil Akhir

Aplikasi berhasil mengelola data produk, pelanggan, dan mahasiswa dengan CRUD lengkap serta tampilan responsif yang siap digunakan sebagai dasar portfolio proyek Laravel.
